# Contributing to ComposableDataflowBlocks

Thank you for considering contributing to **ComposableDataflowBlocks**! We welcome contributions from the community to improve, extend, and refine the project. Please read the following guidelines to get started.

## How Can I Contribute?

There are several ways you can contribute:

- **Reporting Bugs:** Help us improve by submitting issues for bugs you find.
- **Suggesting Enhancements:** Share ideas for new blocks, features, or improvements.
- **Submitting Pull Requests:** Fix bugs, add features, improve documentation, or enhance code quality.
- **Writing Tests:** Help us with code coverage and reliability.

## Getting Started

1. **Fork the Repository**
   - Go to [CounterpointCollective/ComposableDataflowBlocks](https://github.com/CounterpointCollective/ComposableDataflowBlocks) and click "Fork".
2. **Clone Your Fork**
   - Clone your fork locally:
     ```sh
     git clone https://github.com/<your-username>/ComposableDataflowBlocks.git
     cd ComposableDataflowBlocks
     ```
3. **Set Up the Development Environment**
   - Make sure you have [.NET SDK](https://dotnet.microsoft.com/download) installed.
   - Restore dependencies:
     ```sh
     dotnet restore
     ```
   - Build the project:
     ```sh
     dotnet build
     ```
   - Run tests:
     ```sh
     dotnet test
     ```

## Making Changes

- Work on a new feature branch (`git checkout -b feature/my-awesome-feature`).
- Follow existing code style and add comments where appropriate.
- Ensure all tests pass before creating a pull request.
- If you add functionality, write and update tests as needed.

## Submitting a Pull Request

1. **Commit Your Changes**
   - Describe what you changed and why.
2. **Push to Your Fork**
   ```sh
   git push origin feature/my-awesome-feature
   ```
3. **Open a Pull Request**
   - Go to your fork’s GitHub page and click "Compare & pull request".
   - Fill out the PR template, describing your changes and linking any relevant issues.

## Code Review and Merging

All submissions will be reviewed. Feedback may be given and changes may be requested before your contribution can be merged.

- Ensure all automated checks pass.
- Address reviewer comments promptly.
- Once approved, your pull request will be merged!

## Community Standards

- Be respectful and constructive in comments and discussions.
- Adhere to GitHub’s [Community Guidelines](https://docs.github.com/en/site-policy/github-terms/github-community-guidelines).
- No harassment or inappropriate behavior will be tolerated.

## Questions?

If you have any questions, open an issue or join the discussion in the repository.

Thank you for helping make ComposableDataflowBlocks better!
